# NAMM v1.0.3 — UI Redesign Spec

## Goal

Replace the YACL-based step editor with a custom Minecraft Screen to enable inline editing, eliminate unnecessary sub-menus, and make the mod feel snappy and unbloated. Macro list and Profiles stay in YACL with targeted fixes.

## Architecture

- **Macro list + Profiles + Import/Export** (YACL) — stays in YACL as the main config screen opened by ModMenu. The "Edit Steps" button on each macro opens the custom step editor.
- **StepEditorScreen** (custom) — per-macro timeline with inline editing, replaces the old YACL step editor screen.

ModMenu entrypoint opens the YACL config screen (unchanged). The step editor is the only custom screen.

---

## Macro List (YACL — unchanged structure)

The macro list stays in YACL as-is (Name, Playback Mode, Enabled, Trigger Key, Edit Steps, Delete per macro). The "Edit Steps" button opens the new custom StepEditorScreen. No changes to the macro list layout.

---

## StepEditorScreen

Custom `Screen` subclass. The core of the redesign.

### Header area
- **Macro name** — editable `EditBox`, updates on blur/Enter
- **Playback Mode** — `CycleButton` widget cycling through PlaybackMode values
- **Enabled** — toggle button (moved from macro list to here)
- **Back** button — returns to MacroListScreen

### Step list
Scrollable area below the header. Each step is a compact row (~24px) with a 1px gray divider line between steps.

#### Row layout
```
[1.] [Step description................] [↑] [↓] [X]
```
- Step number on the left
- Step description fills the middle
- ↑/↓ arrows and X delete button on the right side
- ↑ hidden on first step, ↓ hidden on last step

#### Input steps (KEY_PRESS, KEY_RELEASE, MOUSE_CLICK, MOUSE_RELEASE)
- Display: `"Key Press: W"` or `"Mouse Click: Left Click"`
- **Click the key name portion** to enter recording mode:
  - Text changes to `"Press a key..."` with highlighted background
  - Next key/mouse event captures the input
  - Row returns to normal display with updated key name
  - **Re-capture preserves the original action type** (press vs release). Only the key code and key-vs-mouse flag are updated. E.g., re-capturing a KEY_RELEASE step changes the key but keeps it as a release.
- Press/release pairs created by "Add Input" are independent steps. Users can freely delete, reorder, or re-capture each one separately.

#### Recording mode (modal)
- `recordingStepIndex` flag tracks which step is being recorded (-1 = none)
- While recording, `keyPressed()` and `mouseClicked()` are intercepted to capture the input instead of dispatching to widgets
- Escape cancels recording (does NOT close the screen)
- All other widget clicks suppressed during recording

#### Delay steps
- Display: `"Delay: 50ms"`
- **Click the value** to edit inline:
  - An `EditBox` appears in-place over the "50" portion
  - Type a number, press Enter or click away to confirm
  - Minimum 20ms enforced
  - Display updates immediately

### Add Step
- **"+ Add Step"** button at the bottom of the step list
- Clicking it shows a small popup overlay with two options:
  - **"Input"** — adds a new input step in recording mode (press + release pair)
  - **"Delay"** — adds a new delay step (default 50ms)
- The popup is a minimal overlay rendered on top of the step list, not a new screen

### Rendering
- Use `GuiGraphics` for custom rendering
- Divider lines: `graphics.fill(x, y, x+width, y+1, 0x44FFFFFF)` between rows
- Recording mode: highlighted row background (subtle color change)
- Scrolling: track scroll offset, render only visible rows, mouse wheel support
- **Scissor clipping**: use `graphics.enableScissor(left, top, right, bottom)` around the step list area to prevent rows from rendering over the header and bottom bar
- **Empty state**: when no steps exist, show centered text "No steps yet — click + Add Step below"

### Persistence
All changes (reorder, add, delete, edit) are saved immediately via `NammConfig.getInstance().save()`. No unsaved-changes state needed.

### Key capture
- `KeyCaptureScreen.java` is kept for the YACL trigger key button (macro list still uses it)
- StepEditorScreen handles inline recording directly via its own `recordingStepIndex` state — no need for a shared helper since only one custom screen uses inline capture

## Profiles (YACL — targeted fixes)

Profiles stay in YACL. Two changes:

### Fix 1: Don't navigate away on activate
When user clicks "Activate" on a profile, currently calls `setScreen(buildMainScreen(parent))` which navigates to the Macros tab. Instead: set `activeProfileName`, call `save()`, and rebuild the YACL screen but navigate to the Profiles category. Use `setScreen(createConfigScreen(parent))` — the user lands back on the config screen. Since YACL doesn't support navigating to a specific category programmatically, this is acceptable (user clicks the Profiles tab again if needed). Alternatively, build a standalone profiles-only YACL screen that the activate button rebuilds.

### Fix 2: Search filter inside each profile
Each profile's `OptionGroup` gets a `StringController` "Search" field at the top with a "Filter" `ButtonOption` next to it. The macro tick boxes below are filtered to only show macros whose name contains the search string (case-insensitive).

Implementation: the search string is stored per-profile in a transient `HashMap<String, String>` on NammConfig (not persisted). When the user clicks "Filter", the YACL screen is rebuilt via `setScreen()` with only matching macros shown. This avoids the YACL limitation of not being able to dynamically hide options — we rebuild the screen on explicit filter action rather than on every keystroke.

## Files to Create/Modify

### New files
- `src/main/java/com/namm/config/StepEditorScreen.java` — custom step editor screen

### Modified files
- `src/main/java/com/namm/config/NammConfig.java` — remove `buildStepEditorScreen()` (replaced by StepEditorScreen). Update `buildMacrosCategory()` "Edit Steps" button to open `StepEditorScreen` instead. Update `buildProfilesCategory()` for no-navigate-away fix and search filter.
- `src/main/resources/assets/namm/lang/en_us.json` — update translations

### Unchanged files
- `src/main/java/com/namm/NammModMenuIntegration.java` — still returns NammConfig YACL screen
- `src/main/java/com/namm/config/KeyCaptureScreen.java` — still used for trigger key capture in YACL macro list

### Unchanged files
- All model classes (Macro, MacroStep, MacroProfile, ActionType, PlaybackMode)
- MacroSerializer, MacroExecutor, InputSimulator, TriggerKeyHandler, KeyNames
- Mixin, build.gradle, fabric.mod.json

### Note on deprecated fields
`MacroStep.delayBeforeMs` and `delayAfterMs` are legacy fields kept for backward-compatible deserialization. They are intentionally not exposed in the new UI. The migration in `MacroSerializer.migrateSteps()` converts them to standalone DELAY steps on load.
