# NAMM v1.0.3 UI Redesign — Implementation Plan

> **For agentic workers:** REQUIRED: Use superpowers:subagent-driven-development (if subagents available) or superpowers:executing-plans to implement this plan. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Replace the YACL step editor with a custom Minecraft Screen featuring inline key recording, inline delay editing, step dividers, and a single "Add Step" button with popup. Fix profile activate navigation and add profile search filter.

**Architecture:** The YACL config screen stays as the main entry point (Macros, Profiles, Import/Export tabs). Only the step editor becomes a custom `Screen` subclass (`StepEditorScreen`). The "Edit Steps" button in the YACL macro list opens this custom screen instead of building a YACL sub-screen.

**Tech Stack:** Java 21, Minecraft 1.21.11 (Mojang mappings), Fabric API, YACL 3.8.1

**Spec:** `docs/superpowers/specs/2026-03-16-v103-ui-redesign.md`

## Task 2: Step list rendering with dividers

**Files:**
- Modify: `src/main/java/com/namm/config/StepEditorScreen.java`

- [ ] **Step 1: Add scroll state and step list constants**

```java
private static final int HEADER_HEIGHT = 40;
private static final int ROW_HEIGHT = 24;
private static final int BOTTOM_BAR_HEIGHT = 30;
private double scrollOffset = 0;
```

- [ ] **Step 2: Render step rows with dividers in render()**

After `super.render()`, render the step list area:
- Calculate visible area: `top = HEADER_HEIGHT`, `bottom = height - BOTTOM_BAR_HEIGHT`
- Enable scissor: `graphics.enableScissor(0, top, width, bottom)`
- For each step, compute `rowY = top + (i * ROW_HEIGHT) - (int)scrollOffset`
- Skip rows outside visible area
- Draw step number: `graphics.drawString(font, (i+1) + ".", 10, rowY + 6, 0xFFFFFF)`
- Draw step description: `graphics.drawString(font, step.getDisplaySummary(), 30, rowY + 6, 0xFFFFFF)`
- Draw divider line after each row: `graphics.fill(5, rowY + ROW_HEIGHT - 1, width - 5, rowY + ROW_HEIGHT, 0x44FFFFFF)`
- Disable scissor: `graphics.disableScissor()`
- Empty state: if no steps, draw centered "No steps yet" text

- [ ] **Step 3: Add mouse scroll support**

Override `mouseScrolled(double mouseX, double mouseY, double horizontalAmount, double verticalAmount)`:
```java
int maxScroll = Math.max(0, macro.getSteps().size() * ROW_HEIGHT - (height - HEADER_HEIGHT - BOTTOM_BAR_HEIGHT));
scrollOffset = Math.max(0, Math.min(maxScroll, scrollOffset - verticalAmount * 10));
return true;
```

- [ ] **Step 3: Override keyPressed to capture during recording**

```java
@Override
public boolean keyPressed(KeyEvent keyEvent) {
    if (recordingStepIndex >= 0) {
        MacroStep step = macro.getSteps().get(recordingStepIndex);
        step.setKeyCode(keyEvent.key());
        // Update mouse flag but preserve press/release action type
        if (step.isMouse()) {
            // Was mouse, now keyboard — update action type to keyboard equivalent
            step.setMouse(false);
            if (step.getActionType() == ActionType.MOUSE_CLICK) step.setActionType(ActionType.KEY_PRESS);
            if (step.getActionType() == ActionType.MOUSE_RELEASE) step.setActionType(ActionType.KEY_RELEASE);
        }
        NammConfig.getInstance().save();
        recordingStepIndex = -1;
        return true;
    }
    if (keyEvent.key() == org.lwjgl.glfw.GLFW.GLFW_KEY_ESCAPE) {
        if (recordingStepIndex >= 0) { recordingStepIndex = -1; return true; }
    }
    return super.keyPressed(keyEvent);
}
```

- [ ] **Step 4: Override mouseClicked to capture mouse during recording**

In `mouseClicked`, before all other handling:
```java
if (recordingStepIndex >= 0) {
    MacroStep step = macro.getSteps().get(recordingStepIndex);

        if (step.getActionType() == ActionType.KEY_PRESS) step.setActionType(ActionType.MOUSE_CLICK);
        if (step.getActionType() == ActionType.KEY_RELEASE) step.setActionType(ActionType.MOUSE_RELEASE);
```

**Files:**

- [ ] **Step 1: Add delay editing state and EditBox**

- [ ] **Step 2: Click delay value to start editing**

In `mouseClicked`, when clicking the description area of a DELAY step:
```java
if (step.getActionType() == ActionType.DELAY && clickX > 30 && clickX < width - 70) {
    editingDelayIndex = clickedIndex;
    int rowY = HEADER_HEIGHT + (clickedIndex * ROW_HEIGHT) - (int) scrollOffset;
    delayEditBox = new EditBox(this.font, 80, rowY + 2, 60, 18, Component.literal("Delay"));
    delayEditBox.setValue(String.valueOf(step.getDelayMs()));
    delayEditBox.setFilter(s -> s.matches("\\d*"));
    delayEditBox.setResponder(val -> {}); // commit on Enter/blur
    addRenderableWidget(delayEditBox);
    delayEditBox.setFocused(true);
    return true;
}
```

- [ ] **Step 3: Commit delay edit on Enter or click-away**

In `keyPressed`, when editingDelayIndex >= 0 and key is Enter:
```java
if (editingDelayIndex >= 0 && keyEvent.key() == GLFW.GLFW_KEY_ENTER) {
    commitDelayEdit();
    return true;
}
```

In `mouseClicked`, if editingDelayIndex >= 0 and click is outside the EditBox area, call `commitDelayEdit()` before processing the click.

```java
private void commitDelayEdit() {
    if (editingDelayIndex >= 0 && delayEditBox != null) {
        try {
            int value = Integer.parseInt(delayEditBox.getValue());
            macro.getSteps().get(editingDelayIndex).setDelayMs(value); // clamps to min 20
        } catch (NumberFormatException ignored) {}
        NammConfig.getInstance().save();
        removeWidget(delayEditBox);
        delayEditBox = null;
        editingDelayIndex = -1;
    }
}
```

- [ ] **Step 4: Render the EditBox over the delay value**

In the render loop, when `i == editingDelayIndex`: skip drawing the description text (the EditBox renders itself via `super.render()`). Reposition the EditBox Y if scroll changed.
